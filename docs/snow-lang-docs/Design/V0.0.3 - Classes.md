# Details and Design
## Type Inspection
- The `__typedesc` object
	- `fn isClass(): bool`: Returns true if the type is a class
### Lifetime Safety 
- Rule 1:
	1) Classes cannot hold references as member variables.
		```
		let x = SomeClass();
		{
			let y = AnotherClass();
			x.mY = ref y; // Assume that x has a variable for AnotherClass type called mY
		} // y goes out of scope here, x.mY contans a reference to this variable that is now pointing at invalid memory
		// The lifetime of x is greater than y in this situation
		```
		- Due to this, classes cannot store references of any kind as member variables. That is a pretty big issue though, how do you hold other classes that are shared between multiple objects? Instead of a raw reference, it needs to be a smart reference such as a `sref` (shared reference)
## Classes
- Classes enclose functionality and data
- Classes are comprised of 3 things, operators, functions, and variables
- Classes are organized however you wish
	```
	class name {
		pub mut x : f32;
		mut y : i32 = 0;
		pub let z : f32 = 1.0;
		static pub let w : f32 = 1.0;
		
		pub init(mut self) {}
		init(mut self, x : f32, y : f32) { self.x = x; self.y = y; }
		pub init(self): copy Self {}
		pub init(mut self): move Self {}
		pub deinit(mut self) {}
		
		pub fn funcA(self) {}
		fn funcB(mut self) {}
		pub fn funcC() {}
		static fn funcD() {}

		pub postop++(mut self) {}
		pub preop++(mut self) {}
		pub binop*(self, num : i32) {}
		pub asop(self): (i32, i32, i32) {}
	}
	```
	- Anything that is public (has the `pub` keyword) can be used outside the class. Everything not containing that can only be used inside this class's functions
	- Variables
		- `x` is a public mutable `f32` that is not initialized
		- `y` is a private mutable `i32` that is default initialized to `0`
		- `z` is a public immutable `f32` that is default initialized to `1.0`
		- `w` is a public immutable `f32` that is default initialized to `1.0` and is statically accessible (`name.w` can be used, it is a type variable not instance)
		- The first three variables can only be accessed on an instance, the last variable can ONLY be accessed on the type. Static variables are always defined with the `static` keyword. Static variables MUST be default initialized. They dont have to be immutable, and they dont have to be public. If the static variable is mutable, then it must be atomically secure which wont be implemented until V0.2.0. Therefore, for now, they must be immutable.
		- Classes cannot store `ref` objects as variables. This causes lifetime issues, so it is prohibited.
		- All variables must be assigned a default value, or are assigned in all initializers
	- Initializers
		- Initializers are called when this class is being created. Deinitializer when it is being destroyed. Initializers may have access to uninitialized variables on the class. The initializer MUST initialize these variables with a value, but if they are accessed before being assigned, then it is an error just as if you defined a variable in a function without setting it and tried to use it.
		- `pub init(self){}` is a public "default initializer" which is generated by the compiler if all variables are default initialized. If not, this and/or another initializer is required. In the case above, all initializers must set `x`. The default initializer does not need to be public. All non-copy non-move initializers take the first argument as `mut self`
		- `init(mut self, x : f32, y : f32){}` is a private initializer. The first argument is never needed to be passed just like functions
		- `pub init(self): copy Self{}` is a copy initializer. No other copy initializers can be made other than this and a variant that takes `mut self`. They are always public. This initializer returns a new `Self` object that is a copy of this one. This function is compiler generated if all variables of this class can be copied, and this class and none of the variables of the class have the `NoCopy` attribute. Only 1 of these kinds of initializers can be on the class. If `NoCopy` attribute is on this class, this initializer is not allowed.
		- `pub init(mut self): move Self{}` is a move initializer. No other move initializers can be made other than this and a variant that takes `self`. They are always public. This initializer returns a new `Self` object that the current object moves it's memory into. This function is compiler generated if all variables of this class can be moved, and this class and none of the variables of the class have the `NoMove` attribute. Only 1 of these kinds of initializers may be on the class. If `NoMove` attribute is on this class, this initializer is not allowed.
		- `pub deinit(mut self) {}` is a deinitializer. This function is called just before the class instance is destroyed, allowing you to clean up any data on the class manually if you need. This signature is the only allowed.
	- Functions
		- The first variable of all non-static functions must either be `self` or `mut self`. This is how you access the variables on the instance of the object. Some languages use `this` as the keyword for it. Static functions do not have this first parameter be `self`, this means it is only callable from the type itself `className.functionName()`
		- The `Self` type represents this class's type without needing to actually type it out. `self` is actually just sugar for `self : ref Self` and `mut self` sugar for `self : ref mut Self`
		- `pub fn funcA(self) {}` is a public function which uses an immutable self (cannot modify variables)
		- `fn funcB(mut self) {}` is a private function which uses a mutable self (can modify variables)
		- `pub fn funcC() {}` is a public static function
		- `static fn funcD() {}` is the same as `funcC` but uses the `static` keyword for better clarity
	- Operators
		- Operators are special functions which are invoked by using symbols around the type. The following symbols can be used in 1-4 combinations `!#%^&*-+=/|<>~`. Illegal combinations: `/*`, `*/`, `//`, `///`, `//!`, `!//`. Additionally `postop()(...)` and `postop[](...)` can be used.
		- `pub postop++(mut self) {}` is a postfix operator using `++` symbols. It is called `obj++`
			- This operator can only take the self parameter
		- `pub preop++(mut self) {}` is a prefix operator using `++` symbols. It is called `++obj`
			- This operator can only take the self parameter
		- `pub binop*(self, num : i32) {}` is a binary operator using `*` symbol. It is called `obj * 4`
			- This operator can only take the self parameter and one other parameter.
		- `pub asop(self): Type {}` is a casting operator which is casting type T. It is called `obj as i32`
			- This operator can only take the self parameter and must return a non-optional type
		- There are a few predefined operators that make math possible. There are two things here, specific operators, operator precedence, and operator associativity. The operator precedence shows how the operators are called. The top most of the following list are executed before the bottom (sort of, you start at the bottom and attempt to execute up the list). Operator associativity is saying that you evaluate from the right side to the left when right associative, and the opposite for left.
			- Left: `postop()`, `postop[]`, `asop`
			- Left: All custom `postop` operators
			- Left: `postop++`, `postop--`
			- Right: All custom `preop` operators
			- Right: `preop+`, `preop-`, `preop!`, `preop~`
			- Left: All custom `binop` operators
			- Left: `binop*`, `binop/`, `binop%`
			- Left: `binop+`, `binop-`
			- Left: `binop<<`, `binop>>`
			- Left: `binop<`, `binop>`, `binop<=`, `binop>=`, `binop==`, `binop!=`
			- Left: `binop&`
			- Left: `binop^`
			- Left: `binop|`
			- Left: `binop&&`
			- Left: `binop||`
			- Right: `binop=`, `binop+=`, `binop-=`, `binop*=`, `binop/=`, `binop%=`, `binop<<=`, `binop>>=`, `binop&=`, `binop|=`, `binop^=`
- Friend classes
	- A class can define itself a friend of another class, giving that class access to the friended classes private objects
	```
	class A {
		let x = 0;
		friend class B;
	}
	class B {
		B(self, A a) {
			print(a.x); // safe since B is a friend of A
		}
	}
	```
	- This is one way, in the example above, A is not a friend of B and cannot access B's privates. However both classes can be friends of one another if they wish
	- You can also friend a function by name with `friend fn NAME;` and that function can access the class's privates
