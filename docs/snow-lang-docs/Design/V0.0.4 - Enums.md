# Details and Design
## Type Inspection
- The `__typedesc` object
	- `fn isEnum(): bool`: Returns true if the type is an enum
## Temporary Enum (variant)
- Defined as a type `< i8, i16 >`
- Basically an unnamed enum that accepts only the specified types
- Can be defined anywhere a type is
- Accessing and RTTI (run time type information)
	- Comparisons are done through the `is` keyword (type comparison)
	- Accessors are done through the `as` keyword (caster)
	```js
	fn validUriToString(uri: < (i8, i8, i8, i8, i16), string >): string? {
		if (uri is (i8, i8, i8, i8, i16)) {
			// uri is now considered this type (which is a tuple)
			return `${uri.0}.${uri.1}.${uri.2}.${uri.3}:${uri.4}`;
		}
		else {
			if (parseUri(uri as string)) {
				return uri as string;
			}
			return undefined;
		}
	}
	```
## Enum type (enum)
- Can only store one value, does not have more than one member
- Enums can have methods similar to classes, but uniquely do not support operators
	- Static and otherwise
```
generic<T> enum RustOptional {
	Some : T,
	None // no type specified
}
```
- An enum value does not require a type, but if specified it must be set to a value either by initializer, or default values
	- When no type is specified, the value is its own type accessed solely through the enum name: `RustOptional<T>.None` is the only way to access this value
