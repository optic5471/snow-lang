# Details and Design
## Task
- Wrapper around a function which utilizes threading mechanics, similar to coroutines
- Function wrapped must be `async`
- `task(F)` where F is an async function type for typename
- `let t : task(fn()) = someFunc;`
- Methods
	- `fn state(): TaskState`: Returns the task's state
		- `TaskState` is an enum. `NotStarted`, `Running`, `Paused = typeof(F).yeildType`, `Finished = typeof(F).returnType`
	- `fn await(): typeof(F).returnType`: Blocks the current thread awaiting for the task to complete, yields are immediately resolves, their value is discarded, returns the functions return type when finished
	- `fn isPaused(): bool`: Returns true if paused
	- `fn isFinished(): bool`: Returns true if finished
	- `fn isStarted(): bool`: Returns true if the task has been executed at least once (can be true if finished)
	- `fn isRunning(): bool`: Returns true if the task is currently executing (on a different thread of course)
	- `fn execute(): enum { typeof(F).yeildType, typeof(F).returnType }`: Executes the task until a yield or return is called
## Lifetime Safety
- Rule 3:
	3) Variables must be moved, copied, or atomically secured to be used in threads or closures. (Not implemented this release, v0.0.5)
		```
		fn func(a : ref i32) {}
		
		// This syntax is not final for threads
		let t;
		{
			let x = 4;
			t = Thread(func, ref x);
		} // x goes out of scope, thread is not. Thread's reference is now invalid memory
		```
		- Threads and closures work similar to one another in that they can reference variables that exist outside of their own internal scope. This can result in invalid memory, so variables must be copied, moved, or atomically secured to be provided to a thread or closure.
		- Atomically securing is done through a class attribute or by wrapping an object in an atomically secured class such as an `asref` (atomic shared reference)
		- There is one caveat to this rule: Closures who do are not referenced outside the scope they are defined in, can capture references as the scope for the closure is well defined that it will not leave
