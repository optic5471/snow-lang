# Goals
- [ ] `unsafe string.ptr(): ptr<[rune]>?`
- [ ] Array
	- [ ] `unsafe array.ptr(): ptr<[T]>?`
- [ ] Static arrays
	- not dynamically sized, sizing and allocation is all up front
- [ ] `unsafe`
- [ ] Name hiding is considered safe, but undesirable. A suggestion warning should be emitted when done
	- `let x = 0; { let x = 3; /* same name, allowed but suggestion */ }`
- [ ] Memory management
	- [ ] The `std.SnowAllocator` is the allocator for the language. All allocations go through this so it is pretty complex and is completely unsafe
	- [ ] Custom allocators
		- [ ] `Allocator` interface
			- [ ] `pub generic<T> fn alloc() : ptr<T>`
			- [ ] `pub alloc(size : i64) : arptr<u8>`
			- [ ] `pub generic<T> fn alloc(size : i64) : arptr<T>`
			- [ ] `pub generic<T> fn dealloc(obj : ptr<T>)`: The deinitializer for T is already called when this is
		- [ ] `GlobalAllocator` object (defaults to `std.SnowAllocator`)
	- [ ] The pointer types
		- [ ] `generic<T> class ptr`
			- [ ] `pub fn deref(): T?`
			- [ ] `pub fn isNull(): bool`
			- [ ] move initializer, this is a shallow move
			- [ ] copy initializer, this is a shallow copy
			- [ ] `pub fn deepCopy(): ptr<T>`, this is not a copy of the pointer but of the underlying object to a new pointer type
			- [ ] `pub fn deepMove(): ptr<T>`, this is not a move of the pointer but of the underlying object to a new pointer type
			- [ ] `pub fn release(a : ref mut Allocator)`, sets this pointer to null and releases the memory to the allocator
			- [ ] Does NOT implement a deinitializer, the allocator must unset the pointer. In debug mode, there will be a deinitializer which if this still has memory will panic
		- [ ] `generic<T> class arptr`
			- [ ] Same as `ptr`
			- [ ] `pub fn len(): u64`
			- [ ] `pub postop[](index : u64): ref T?`
			- [ ] `pub postop[](index : u64): ref mut T?`
			- [ ] `pub resize(index : u64)`
- [ ] Standard library
	- [ ] The `PanicHandler` type
		- pure static implementation
		- [ ] `pub OnPanic : (fn(msg : string))? = undefined`: Will be called when a panic happens
			- The abort will happen, this just lets you do one more thing before it does
		- A way to get the current call stack with friendly information
	- [ ] `shared<T>`
		- A shared object which allows multiple references to the same value but is not atomically secure
		- With this, two classes can have references to each other as variables
		- Allocates the object not on the stack, but on the heap
		- Object is ref counted, each copy increments the count and each deinit decrements
			- When the count reaches zero, the underlying value is destroyed and the memory released
		- [ ] `pub fn get(mut self): ref mut T`
		- [ ] `pub postop->(mut self): ref mut T`

# Details and Design
## Unsafe
- Can be applied to an interface, enum, class, function, variable, module and as a scope `unsafe {}`
	- When on an interface, enum, class, function, or module; all of the contents in those are unsafe
	- When on a variable, only using the variable is unsafe
	- When as a scope, all contained is unsafe
- The unsafe keyword is used to mark something as being inherently unsafe to do. The language will enforce that only unsafe things can be used by unsafe things. This keeps safe code deliberately incapable of calling unsafe code
	- Unsafe code can call safe code
- To make an something that is unsafe, usable by safe code, you must not make the class or function unsafe, but rather do the unsafe thing inside the scope of the function. This does hide unsafe code from callers, but it is on the end user to ensure that their unsafe code is being handled correctly. Much of the standard library may use unsafe code
- Possibly make a compiler argument `--no-unsafe` which errors on any usage of unsafe not accompanied by a `safe` attribute with an explaination
