# Details and Design
## Type Inspection
- The `__typedesc` object
	- `fn isInterface(): bool`: Returns true if the type is an interface
- The `implements` keyword
	- In where clauses is used to return true if the type implements the interface `type implements interface`
## Interfaces
- Casting a class to an interface that it implements is always possible without a casting function being defined on the class
- Interfaces define a set of functionality that a class must implement
- Classes can only implement interfaces, there is no inheritance in snow
- All functions on an interface must be public
- Interfaces can define operators
- When `Self` is used in an interface, it refers to the actual class not the interface type
- Variables can hold interface types, however you cannot instantiate one. Instead you can reference, copy, move, and assign a class which implements said interface. All calls to the interface are then redirected to the class
- You can check if an interface variable is a class with `x is y`, where x is the variable and y is the class type.
	- With this you can cast from an interface to a class type. The compiler requires that you check for the interface being that type before casting to it using the `as` keyword
```
interface Factory {
	static pub fn make() : Self;
}
interface A {
	pub fn doThing();
	pub binop++(mut self, x : i32);
}

class Foo implements A, Factory {
	static pub fn Factory.make() : Self { return Foo(); }
	pub fn A.doThing(self) {}
	pub binop A.++(mut self, x : i32) {}
}

let f = Foo.make();
let fa : A = copy f;
fa.doThing();
fa ++ 10;
if (fa is Foo) {
	let k = fa as Foo;
}
```
