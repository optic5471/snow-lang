# Details and Design
## Type Inspection
### Is Keyword
- This keyword allows you to compare that the type of a variable or generic is another type
- `T is SomeClass`, `T is K` (both generic types)
- Return is a compiler constant true or false, usable in where statements
### Has Keyword
- This keyword allows you to ask if the type has a function name or variable name
- The return type is a compiler constant optional type description
- `T has funcName`
- Can only `has` for a public thing on the object
### Typeof Keyword
- This keyword returns a constant type description object from a value
- `typeof(T).name()`, `typeof(someVariable).name()`
### Type Description Object (`__typedesc`)
- This object does not formally exist in anyway that is instantiable by users. The compiler generates this object when a type description is needed
- The type description object has the following API, all functions and variables are compiler constants, so they can be used in where statements and are evaluated at compile time
	- `fn isBuiltIn(): bool`: Returns true if the type is a compiler built-in type
	- `fn isNumber(): bool`: Returns true if the type if an integral or floating point number
	- `fn isFunction(): bool`: Returns true if the type is a function
	- `fn isTuple(): bool`: Returns true if the type is a tuple type
	- `fn isArray(): bool`: Returns true if the type is an array type
	- `fn isMap(): bool`: Returns true if the type is a map type
	- `fn isOptional(): bool`
	- `fn isImmutable(): bool`
	- `fn isGeneric(): bool`
	- `fn getUnderlyingTypes(): [__typedesc]`: Returns the tuple types, array type, function argument types, or map key and value types, can be empty if not one of those types
	- `fn getReturnType(): __typedesc?`: Returns the return type of a function or empty if not a function
