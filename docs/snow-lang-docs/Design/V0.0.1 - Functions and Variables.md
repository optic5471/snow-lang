# Details and Design
## Built-In Types
### Integrals
- By default, all integrals are `i32` or `i64` or `u64` if the default value is too large for the previous
	- Exceeding `u64` as raw input results in error
- By default, all floating points are `f32` or `f64` if the number of fractional digits is greater than 6
- `i8`, `i16`, `i32`, `i64` are signed
- `u8`, `u16`, `u32`, `u64` are unsigned
- Statics
	- `MinVal`: The minimum value
	- `MaxVal`: The maximum value
	- `BitLen`: The number of bits
- Tokenization:
	- Raw numbers are read as base 10: `1234`
	- Prefixes
		- `0u`: unsigned value of `u32` or `u64` depending on size
		- `0i`: signed value of `i32` or `i64` depending on size
		- `0b`: restricts to 0 or 1 and reads as binary into `u32` or `u64` depending on size
		- `0o`: restricts to 0-7 and reads as octal into `u32` or `u64` depending on size
		- `0x`: restricts to 0-9A-Fa-f and reads as hex into `u32` or `u64` depending on size
	- Values may include `_` and '\`' in the middle of them to provide separation. This must come after the first number or after prefix
### Floating
- `f32` and `f64`, both are signed
- `inf` is a token which represents either a `f32` or `f64` infinite value depending on context
	- This can be negative
- `nan` is a token which represents either a `f32` or `f64` "not-a-number" value depending on context
	- This cannot be negative
	- Usage of nan numbers is a warning
	- Using nan to do anything results in a nan output
	- nan cast to integral is 0
		- This can cause runtime errors with a compiler flag set, useful for debugging purposes
- Statics
	- `Epsilon`: `0.00001` in `f32` and `0.000000001` in `f64`
	- `fn near(T a, T b, T e = T.Epsilon): bool`: Returns true if `a` is `e` within `b`
- Tokenization
	- Raw numbers `1234.0` are f32 depending on number of fractional digits
	- `1.1e2` and `1.1-e10` are allowed as scientific notation
### Rune
- A single character of a string in ascii. 1 byte in length
- Declared as a single quote and a string of length 1
	- `'c'` and `'\n'`
	- `"c"` and `"\n"`
- Escapes with `\`
	- `\n`: newline
	- `\'`: single quote (only required in single quoted rune)
	- `\"`: double quote (only required in double quoted rune)
	- `\\`: back slash
	- `\r`: carriage return
	- `\t`: horizontal tab
- Statics
	- `fn isAlpha(): bool`: Returns true if the character is an alphabet character (a-zA-Z)
	- `fn isNumeric(): bool`: Returns true if the character is a number (0-9)
	- `fn isWhitespace(): bool`: Returns true if the character is a space, tab, newline, or carriage return
	- `fn isLower(): bool`: Returns true if the character is a-z
	- `fn isUpper(): bool`: Returns true if the character is A-Z
	- `fn toLower(): rune?`: Returns the lower case character of A-Z, if not able to convert, returns undefined. If already lower case, returns value.
	- `fn toUpper(): rune?`: Works the same way as toLower, but in reverse
	- `fn toEscaped(): string`: Returns a string of the current character escaped. If the character doesnt require escaping, the string of 1 character is returned. Otherwise the string of of the escaped character is.
	- `fn parse(in: string): rune?`: Parses a single character from the string. If the character is escaped, it will parse the full character. So `\n` will be parsed into the newline character. Returns undefined if the string is empty
### String
- `"hello world"`: Standard string input, characters must be escaped
- `@"hello     world"@`: A What You See Is What You Get (WYSIWYG) string, no escaping is required. Cannot include `"@` as that is the end tag
	- Can include a single character though between the two symbols `@A"hello  world"A@` to change what marks the end.
- \`hello ${variable} \\${v}\` with `variable` equaling `5`, will result in `"hello 5 ${v}`
	- Notice that escaping the `$` allows `${}` to be used in the string
	- This is a format string
	- The value of any variables captured must implement the `Formattable` interface
- Statics
	- `Empty`: The empty string, this can be used to return an immutable reference from functions
	- `fn parse(in: string): string?`: Parses the input string, converting all escaped values to their actual characters. Returns undefined if the string was empty
- Members
	- `fn len(): u64`: Returns the number of characters in the string
	- `fn isEmpty(): bool`: Returns true if the string is empty
	- `fn data(): [rune]`: Returns the data of the string
	- `fn find(delim: rune): u64?` and `fn find(delim: string): u64?`: Finds the first value in the string matching the delimiter. Returns index if found, undefined otherwise.
	- `fn findAll(delim: rune): [u64]` and `fn findAll(delim: string): [u64]`: Finds all matching indices of the delimiter and returns them as an array. Returns empty array if not found
	- `fn findReverse(delim: rune): [u64]` and `findReverse(delim: string): [u64]`: Same as find, but starts search at end of string instead of beginning
	- `fn findNext(delim: rune, index: u64): u64?` and `findNext(delim: string, index: u64): u64?`: Same as find, but starts search at the provided index. Returns undefined if index is greater than string length
	- `fn split(delim: rune): [string]`: Returns an array of strings split by the delimiter, delimiter is not included in split strings.
	- `fn startsWith(delim: string): bool` and `fn startsWith(delim: rune): bool`: Returns true if the string starts with the delimiter
	- `fn endsWith(delim: string): bool` and `fn endsWith(delim: rune): bool`: Returns true if the string ends with the delimiter
	- `fn trim()`: Trims the whitespace from the beginning and end of string
	- `fn trimStart()`: Trims the start of the string of whitespace
	- `fn trimEnd()`: Trims the end of the string of whitespace
	- `fn toEscaped(): string?`: Returns the string using escape sequences instead of actual values, undefined if empty.
	- `fn push(c : rune)`: Adds the rune to the end of the string
	- `fn pop(): rune?`: Removes the last rune from the string and returns it, or undefined if the string is empty
	- `fn remove(index : u64, length: u64 = 1)`: Removes length runes starting at index
	- `fn insert(index : u64, val: rune): u64`: Inserts the new rune at the index. Inserts at end if the index exceeds string length. Returns the index that was inserted at
		- To be clear, this moves any value at the index back
	- `fn clear()`: Clears the string of data
	- `fn first(): rune?`: Returns the first rune in the string if not empty, may be the last as well
	- `fn last(): rune?`: Returns the last rune in the string if not empty, may be the first as well
- Operators
	- `postop[](mut self, index : u64): mut ref rune?`: Returns the rune at the index or undefined if outside the index
	- `postop[](self, index : u64): ref rune?`: Returns the rune at the index or undefined if outside the index
### Array
- Dynamically sized array, static sized arrays are not implemented here, that would be a part of some really deep memory stuff
- `[T]` for typename
- `[1, 2, 3]` for instance
- `let x : [i32] = [1, 2, 3];`
- `[]` empty array of any type
- Types must not have both `UnCopyable` and `UnMoveable` attributes
- Members
	- `fn first(): T?`: Returns the first value in the array, or undefined
	- `fn last(): T?`: Returns the last value in the array, or undefined
	- `fn isEmpty(): bool`: Returns true if the array is empty
	- `fn len(): u64`: Returns the number of elements in the array
	- `fn clear()`: Clears the array to empty
	- `fn insert(startIndex : u64, val: enum { T, [T] }): bool`: Inserts the element(s) at the start index or end of the array if beyond the array bounds (moving anything at the index and beyond back). Returns the index that was inserted at
	- `fn remove(index: u64, length: u64 = 1)`: Removes length elements from the array at index
	- `fn push(val: T)`: Pushes the element to the end of the array
	- `fn pop(): T?`: Removes the last element from the array and returns it. Returns undefined if empty
	- `fn find(val : T): u64`: Compares the values until the first match, returns index of match
	- `fn findAll(val : T): [u64]`: Compares the values for all matches, returns the index of all matches
	- `fn find(comparitor: fn(v:ref T):bool): u64`: Same as find, but calls comparitor instead of using default comparisons
	- `fn findAll(comparitor: fn(v:ref T):bool): [u64]`: Same as find all, but calls comparitor instead of using default comparisons
	- `fn contains(val: T): bool`: Returns true if the value is in the array using standard comparison
	- `fn contains(comparitor: fn(v:ref T):bool): bool`: Returns true if the value is in the array using the comparitor
- Operators
	- `postop[](mut self, index: u64) -> mut ref T?`: Returns the value at the index or undefined
	- `postop[](self, index: u64) -> ref T?`: Same as `postop[]` but returns immutable
### Map
- Key-value pair hash map
- `[K, V]` for typename
- `[(4, '4'), (5, '5')]` for instance
- `[]` empty map of any type
- Keys must implement `Hashable`, Values must not have both `UnCopyable` and `UnMoveable` attributes
- Methods
	- `fn isEmpty(): bool`: Returns true if the map is empty
	- `fn len(): u64`: Returns the number of elements in the map
	- `fn clear()`: Clears the map to empty
	- `fn push(key: K, val: V)`: Adds the value to the map, overwriting if needed
	- `fn pop(key: K): V?`: Removes the value from the map if found and returns it
	- `fn find(key : K): ref V?`: Returns the found value if stored in map
	- `fn find(comparitor : fn(v: (ref K, ref V)):bool): ref V`: Returns the first value if stored in map
	- `fn findAll(comparitor : fn(v: (ref K, ref V)):bool): [ref V]`: Returns all the found values if stored in map
	- `fn contains(key: K): bool`: Returns true if the key is in the map
	- `fn contains(comparitor: fn(v: (ref K, ref V)):bool`: Returns true if a match to this value is found
- Operators
	- `postop[](key: K) -> ref V?`: Returns the value if found
### Tuples
- A way to define structures of pure data (POD)
- Similar to an unnamed class, like unnamed enums are similar to variants
- `()` The empty tuple, cannot be assigned to all tuple types, this will produce warnings if used
- Variable declarations
	- Given the function `fn foo(): (i32, i32)`
	- `let a = foo();`, a is now the tuple type `(i32, i32)`
	- `let (x, y) = foo();`, x is the first value of the tuple, y is the second
		- When defined like this, the receiving tuple must contain space for all values in the tuple. Use `_` to define throw away variables.
		- `let (_, y) = foo();` would be used if you only care about the second value
- `(T1, T2, ...)` for typename
- `(4, 5)` for instance
- `(varA, copy varB, 5)` for instance which is a `(ref typeof(varA), typeof(varB), u32)`
- `var.#` is to access tuple value. Compile time errors are produced when indexing beyond tuple length
	- `var.0`
- Methods
	- `fn len(): u64`: Returns the size of the tuple

### Optional
- `undefined` is the typename for an empty optional
- `let x = undefined;` Untyped, but simply is sugar for `let x;`, its unassigned at this point but unassgined variables can be compared against `undefined`
- `let x : T? = undefined;`
- Does not have any methods, simply a moniker on the type similar to mutability and references
- Optional types must be checked for undefined before use
- `?.` operator can be used with optional values like so:
	- No other operators allowed unless the value is checked for being undefined or not first (compiler error)
```
let x : SomeClass? = undefined;
x?.someFunc(); // sugar for `if (x != undefined) { x.someFunc() }`
```
- Special syntax `if (x)` is also allowed which is sugar to `if (x != undefined)`. Auto casting like this is generally not allowed, but for optionals we can
### Function Type
- Defined as `fn(...): T`, basically a function without a name
	- Or `init(...): T` for an initializer function
	- Or `deit(...)` for a deinitializer function
- There is no way to constrain to a specific type without wrapping in a class with a where clause and generics
	```
	// Possible example
	generic(X = SomeClassName) where(X is SomeClassName)
	class SomeClassNameConstraint {
		alias Func = fn();
	}
	// usage
	let x = SomeClassName();
	let specificFnPtr : SomeClassNameConstraint.Func = x.func;
	specificFnPtr();
	```

## Alias
- Alias allows you to rename a type
- `alias newName = Type;`
- Can be used within a file scope, class, enum, interface, and scope expression
	- scope expression means that this alias is ONLY available during the expression
		- Warning when not used
	- The rest allow you to do look ups based on scope
	```
	class A {
		pub alias X = T;
	}
	A.X
	```
- Note that an alias is like any type declaration that can be public, private, exported, ...
## Variables
- `let x;`
	- This is a variable declaration with no type. The type of this variable will be assigned the first time it is used. Note that ALL uses of the variable MUST conform to one type. You cannot use an if statement to define it as one type or another. 
	- The variable's type is discovered at compile time, that is why it cannot be dynamically set
	- Using `typeof` on the variable does actually work, though it produces a suggestion warning since that is kind of weird
	- `let` creates a immutable variable, the underlying type can use immutable operators and functions. The variable cannot be reassigned
	- Usage of this variable before it is set is a compiler error
- `mut x : u8;`
	- This is a normal variable declaration with not type. Similar to the previous, the value of this variable will be assigned on its first assignment, until then it is invalid to use the variable
	- The main difference between this and the previous declaration is the type being set directly rather than by the compiler
	- `mut` creates a mutable variable, the underlying type can use any operators and functions. The variable can be reassigned
	- Usage of this variable before it is set is a compiler error
- `let x = mut ref y;`
	- The `ref` keyword allows you to create a reference (pointer) to another variable. You can use `x` as if it was `y`
	- The `ref` keyword MUST be on the right side of the assignment. This is because we are declaring that this variable `x` is being assigned to a mutable reference to `y`.
	- The type of `x` is `mut ref i32` if `y` is an `i32`. The odd thing is that this was declared with `let`. This just means that the reference `x` is immutable but the value it points to (`mut ref i32`) is mutable.
	- How do you reassign a reference?
		```
		mut y = 0; // i32
		mut x = mut ref y; // x is `ref i32`
		x = 12; // y is now 12
		mut z = 0;
		x = mut ref z; // this assignment is modifying where the reference is pointing to, x now points at z
		x = 15; // y is still 12, z is now 15
		```
	- Lets look at something more complex
		```
		mut x = 2; // x is `i32`
		mut y = mut ref x; // y is `mut ref i32`
		mut z = ref y; // z is `ref (mut ref i32)`
		let a : ref (ref (mut ref i32)) = z; // the parentheses are not required, but can help
		z = 4; // not allowed, the reference is immutable
		y = 4; // allowed, the reference is mutable
		
		mut xprime = 3;
		mut yprime = mut ref xprime;
		z = ref yprime; // allowed, assigned z to a new reference value
		```
	- `ref` is an immutable ref, the value can use immutable functions and operators
	- `mut ref` is a mutable reference, the value can use all functions and operators
- This sounds very confusing with references. But its actually quite simple if we separate the variable from the value. For example, `mut x = ref y;`. Here we have a mutable variable x. The variable has type `ref i32` which is an immutable reference to an `i32`. The variable is assigned to the address of `y`. 
	- Notice, the variable is mutable, but the reference is not. This allows the variable x to be reassigned, but the value the variable is pointing at is immutable. So `x = 12;` is invalid, as this is assigning a value to `y` through an immutable reference.
	- However, `x = ref z;` is okay as we are reassigning the variable `x`, not the value of `x`. 
	- By separating the variable from the value, we can see how references can be reassigned through `ref` syntax if the variable is mutable.
	- This pattern works for non-references as well. `let x : [i32] = [];` is an immutable variable. However the value is mutable. So we can `x.push(3);` as this is modifying the value, but `x = [];` is not valid since it is reassigning the variable which is immutable.
	- The variable declaration can use `let`, and `mut` declarators
	- The variable value can use `mut`, `ref`, `copy`, and `move` markers
- `copy` keyword
	- The copy keyword will attempt to clone the value into a new value. This is done through a copy initializer, which is automatically generated for all types so long as the type does not contain a type that cannot have a copy initializer. 
	- `init(self): copy Self`: Is the definition of a copy initializer
		- Note this is the only type `copy` can be used on a return type
	- If defined, this class can always be copied
	- If not defined, the compiler will generate one if the type has no `NoCopy` types
	- If you wish to make a type NOT be copiable you can add the attribute `NoCopy` to it
- `move` keyword
	- The move keyword will attempt to move the value into a new value, leaving the last value set to `undefined`. This is done through a move initializer, which is automatically generated for all types so long as the type does not contain a type that cannot have a move initializer. 
	- `init(self): move Self`: Is the definition of a move initializer
		- Note this is the only type `move` can be used on a return type
	- If defined, this class can always be moved
	- If not defined, the compiler will generate one if the type has no `NoMove` types
	- If you wish to make a type NOT be movable you can add the attribute `NoMove` to it
	- Move operations are far more efficient than copy operations due to memory being moved rather than copied. The original variable that contained the data becomes `undefined` and is a compiler error to use again unless reassigned.
- Memory management rules with variables and functions:
	- There are no implicit copy or casting operations in snow, everything is manual
	- To perform a copy, cast, or move operation; you must use the appropriate syntax
	- Due to this, `binop=(self): Self`, `binop=(mut self): Self` are disallowed operators. The copy and move initializers should instead be implemented
	- Functions take arguments by either reference or value. Passing by reference is implicit, passing by value is not as that would be either a copy or move operation. For example:
		```
		fn func(a : [i32], b : mut [i32],
				c : ref [i32], d : mut ref [i32]) {
			// a is passed by immutable value. The value could be copied or moved
			// b is passed by mutable value. The value could be copied or moved
			// c is passed by immutable reference
			// d is passed by mutable reference
		}
		
		let q = [];
		let r = [];
		let x = [];
		let y = [];
		func(
			copy q, // the value of q is copied into the function parameter a
			move r, // the value of r is moved into the function parameter b, r is now undefined
			x, // the value of x is passed by immutable reference to the function parameter c
			mut y, // the value of y is passed by mutable reference to the function parameter d
		);
		```
	- Notice that values must always be passed with a marker `copy` or `move`, but references only require the `mut` marker to pass by mutable reference. This signals to the callee how the memory of the parameters is being used without looking at the function signature.
### Lifetime Safety 
- Lifetime safety is a very important concept for modern languages. Snow handles lifetimes through strict memory movement rules. 
	1) Classes cannot hold references as member variables. (Not implemented this release, v0.0.3)
		```
		let x = SomeClass();
		{
			let y = AnotherClass();
			x.mY = ref y; // Assume that x has a variable for AnotherClass type called mY
		} // y goes out of scope here, x.mY contans a reference to this variable that is now pointing at invalid memory
		// The lifetime of x is greater than y in this situation
		```
		- Due to this, classes cannot store references of any kind as member variables. That is a pretty big issue though, how do you hold other classes that are shared between multiple objects? Instead of a raw reference, it needs to be a smart reference such as a `sref` (shared reference)
	2) Moving an object which has a reference attached
		```
		let a = Thing();
		let b = Obj();
		let c = ref b;
		a.mObj = move b; // Error, variable b has a reference to it, cannot move
		```
		- References imply multiple ownership, since references cannot be stored in objects, they can only be stored on the stack which makes tracking variables having references to them quite simple. Because moving an object would cause the reference to then point at invalid memory, this cannot be done.
	3) Variables must be moved, copied, or atomically secured to be used in threads or closures. (Not implemented this release, v0.0.5)
		```
		fn func(a : ref i32) {}
		// This syntax is not final for threads
		let t;
		{
			let x = 4;
			t = Thread(func, ref x);
		} // x goes out of scope, thread is not. Thread's reference is now invalid memory
		```
		- Threads and closures work similar to one another in that they can reference variables that exist outside of their own internal scope. This can result in invalid memory, so variables must be copied, moved, or atomically secured to be provided to a thread or closure.
		- Atomically securing is done through a class attribute or by wrapping an object in an atomically secured class such as an `asref` (atomic shared reference)
		- There is one caveat to this rule: Closures who do are not referenced outside the scope they are defined in, can capture references as the scope for the closure is well defined that it will not leave
